# Database

## Overview

BookWise uses PostgreSQL with Drizzle ORM. The current uncommitted schema update changes borrowing from a manually maintained availability model to a counter-based inventory model with a generated `available_copies` column.

## Technology Stack

- Database: PostgreSQL (Neon Serverless)
- ORM: Drizzle ORM
- Driver: `@neondatabase/serverless`
- Migrations: Drizzle Kit

## Connection Setup

```typescript
import { drizzle } from "drizzle-orm/neon-http";
import { neon } from "@neondatabase/serverless";
import config from "@/lib/config";

const sql = neon(config.env.databaseUrl);
export const db = drizzle({ client: sql });
```

## Tables

### `users`

```typescript
export const users = pgTable("users", {
  id: uuid("id").notNull().primaryKey().defaultRandom(),
  fullName: varchar("full_name", { length: 255 }).notNull(),
  userAvatar: text("user_avatar"),
  userAvatarFileId: text("user_avatar_file_id"),
  email: varchar("email", { length: 255 }).notNull().unique(),
  universityId: varchar("university_id", { length: 30 }).notNull().unique(),
  password: text("password").notNull(),
  universityCard: text("university_card").notNull(),
  status: STATUS_ENUM("status").default("PENDING"),
  role: ROLE_ENUM("role").default("USER"),
  lastActivityDate: date("last_activity_date").defaultNow(),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow(),
});
```

Notable fields:

- `userAvatar` and `userAvatarFileId` support stored profile images
- `universityId` length is now `varchar(30)`

### `books`

```typescript
export const books = pgTable("books", {
  id: uuid("id").notNull().primaryKey().defaultRandom(),
  title: varchar("title", { length: 255 }).notNull(),
  author: varchar("author", { length: 255 }).notNull(),
  genre: varchar("genre", { length: 255 }).notNull(),
  rating: real("rating").notNull().default(0),
  totalCopies: integer("total_copies").notNull().default(1),
  borrowedCount: integer("borrowed_count").notNull().default(0),
  reservedCount: integer("reserved_count").notNull().default(0),
  availableCopies: integer("available_copies")
    .generatedAlwaysAs(sql`total_copies - borrowed_count - reserved_count`)
    .notNull(),
  description: text("description").notNull(),
  coverColor: varchar("cover_color", { length: 7 }).notNull(),
  coverUrl: text("cover_url").notNull(),
  videoUrl: text("video_url").notNull(),
  summary: text("summary").notNull(),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow(),
}, (table) => ({
  availableCopiesIdx: index("available_copies_idx").on(table.availableCopies),
}));
```

Inventory semantics:

- `borrowedCount`: approved active loans
- `reservedCount`: pending requests holding a copy
- `availableCopies`: generated by PostgreSQL and never written directly by application code

This means the source of truth for inventory is the pair of counters, not the displayed availability column.

### `borrow_records`

```typescript
export const borrowRecords = pgTable("borrow_records", {
  id: uuid("id").notNull().primaryKey().defaultRandom(),
  userId: uuid("user_id").references(() => users.id).notNull(),
  bookId: uuid("book_id").references(() => books.id).notNull(),
  borrowDate: timestamp("borrow_date", { withTimezone: true })
    .defaultNow()
    .notNull(),
  dueDate: date("due_date").notNull(),
  returnDate: date("return_date"),
  borrowStatus: BORROW_STATUS_ENUM("borrow_status")
    .default("PENDING")
    .notNull(),
  reservedAt: timestamp("reserved_at", { withTimezone: true }),
  dismissed: integer("dismissed").default(0).notNull(),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow(),
}, (table) => ({
  bookStatusIdx: index("book_status_idx").on(table.bookId, table.borrowStatus),
  reservedAtIdx: index("reserved_at_idx").on(table.reservedAt),
}));
```

Notable fields:

- `reservedAt` marks when a `PENDING` request claimed a copy
- `borrowStatus` now includes `REJECTED`
- `reservedAtIdx` supports stale reservation cleanup
- `bookStatusIdx` supports book/status lookups during borrowing flows

## Relationships

```text
users (1) <- (N) borrow_records (N) -> (1) books
```

- A user can have many borrow records
- A book can have many borrow records
- Each borrow record belongs to exactly one user and one book

## Borrowing Data Rules

The current implementation relies on these invariants:

1. `available_copies` must equal `total_copies - borrowed_count - reserved_count`
2. `PENDING` requests consume `reserved_count`
3. `BORROWED` records consume `borrowed_count`
4. `REJECTED`, `RETURNED`, and `LATE_RETURN` consume neither counter
5. `reserved_at` is used to expire stale reservations after 15 minutes

## Migration `0004_shocking_lady_deathstrike`

The uncommitted migration introduces the new inventory model.

Key changes:

- adds `REJECTED` to `borrow_status`
- adds `reserved_at` to `borrow_records`
- adds `user_avatar` and `user_avatar_file_id` to `users`
- adds `borrowed_count` and `reserved_count` to `books`
- backfills:
  - `borrowed_count` from legacy `BORROWED` records
  - `reserved_count` from legacy `PENDING` records
- drops the old writable `available_copies` column
- recreates `available_copies` as a generated stored column

## Query Patterns

### Reserve a copy atomically

```typescript
const [updatedBook] = await db
  .update(books)
  .set({ reservedCount: sql`${books.reservedCount} + 1` })
  .where(
    and(
      eq(books.id, bookId),
      sql`${books.reservedCount} + ${books.borrowedCount} < ${books.totalCopies}`,
    ),
  )
  .returning({
    availableCopies: books.availableCopies,
    reservedCount: books.reservedCount,
    borrowedCount: books.borrowedCount,
  });
```

### Approve a request

```typescript
await db.update(books).set({
  reservedCount: sql`GREATEST(0, ${books.reservedCount} - 1)`,
  borrowedCount: sql`${books.borrowedCount} + 1`,
});
```

### Return a book

```typescript
await db.update(books).set({
  borrowedCount: sql`GREATEST(0, ${books.borrowedCount} - 1)`,
});
```

## Performance Notes

- `available_copies_idx` supports availability-heavy filtering and sorting
- `book_status_idx` helps status-scoped borrow queries
- `reserved_at_idx` helps reservation-expiry scans
- Generated `available_copies` avoids repeated recomputation in application code while preserving consistency
